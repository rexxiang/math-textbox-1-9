# 数学自学教材内容创作约定

> 本文件是 `AGENTS.md` 的**实现细节补充**。
> `AGENTS.md` 定义顶层合同：读者骨架、语义锚点、分支家族、难度星级、禁用 emoji、`make check` 作为校验入口。本文件仅记录衍生的操作规则，不重复上述合同本身。
> （项目定位中提到的 AoPS / Beast Academy 风格——探索先于讲解、概念驱动、螺旋式深化——是教学取向描述，不是结构规则。）

---

## 全书读者骨架

| 层级 | 结构 |
|---|---|
| 全书 | 导读 → 公共基础 → 共通桥梁 → 分支进阶 → 终章综合 |
| 每分支（代数 / 几何 / 函数 / 数据） | 分支入口 recap → 分支知识点章 → 分支总结 |

## 当前章节目录

> 章节图谱的权威来源是 `docs/knowledge-graph/book-graph.yaml` 与 `typst/main.typ`；下表仅为概览，如与源文件不符以源文件为准。

| 段 | 子目录 | 名称 | 性质 |
|---|---|---|---|
| 0 | `typst/0-orientation/` | 导读与使用导航 | 非知识点叙事 |
| I | `typst/1-public-foundation/` | 公共基础（数、量、形、位置、关系、公共基础总检） | 知识点章节 + 一次总检 |
| II | `typst/2-common-bridges/` | 共通桥梁（比 / 数轴 / 坐标 / 字母 / 方程 / 平方根 / 桥梁 readiness contract） | 桥梁章节 |
| IIIA | `typst/3a-algebra-symbols/` | 代数与符号 | 分支进阶 |
| IIIB | `typst/3b-geometry-proof/` | 几何与证明 | 分支进阶 |
| IIIC | `typst/3c-functions-change/` | 函数与变化 | 分支进阶 |
| IIID | `typst/3d-data-uncertainty/` | 数据与不确定性 | 分支进阶 |
| IV | `typst/4-capstone/` | 终章综合 | 非知识点综述 |

**文件命名与结构**：每章一个子目录，目录名直接表达排序（例 `typst/2-common-bridges/05-equation-balance-inverse-operations/`）；章内文件固定顺序：

- `00-overview.typ`：章首页（title + 语义锚点 + 难度星级 + 核心问题 + 前置旧工具 + 发明树位置 + 回补入口；桥梁章额外写"这座桥通向哪些后续分支"）
- `01-*.typ` … `0N-*.typ`：知识点节 / 桥梁小节
- `99-review.typ`：章末回顾（速查卡 + 混合自测 + 参考答案 + 螺旋复习 + 可选"高中桥"）

## 六阶段发明链：适用范围

六阶段链**只在**知识点章节的 `##` 小节和桥梁章节的小节里使用，顺序不可颠倒：

| # | 阶段 | Typst 函数 | 视觉 | 作用 |
|---|---|---|---|---|
| 1 | **历史困境** | `#crisis[...]` | 红色框 | 以真实或历史困境引出数学需求 |
| 2 | **探索发现** | `#discovery[...]` | 橙色框 | 引导学生重演发现过程 |
| 3 | **试一试** | `#tryit[...]` | 蓝色虚线框 | 例题前主动尝试（先试再读） |
| 4 | **工具蓝图** | `#blueprint[...]` | 蓝色实框 | 正式概念 / 公式 / 定理 |
| 5 | **常见陷阱** | `#pitfall[...]` | 红黄警示框 | ✗ / ✓ 对比展示高频错误 |
| 6 | **工具磨砺** | `#mastery[...]` | 绿色框 | 基础 → 应用 → 挑战三级练习 |

> 困境在先，工具在后。`#tryit` 出现在 `#blueprint` **之前**——学生先尝试，再看正式工具。

**不适用六阶段链的章节**（按各自目的展开，不得硬套链条造假危机）：

- Part 0 导读的所有章节
- 公共基础总检（`1-public-foundation/10-foundation-readiness-check/`）
- 共通桥梁 readiness contract（`2-common-bridges/07-bridge-readiness-contracts/`）
- 各分支总结章
- Part IV 终章综合

这些元章节使用 `#blueprint` 陈述要点、`#side-hack` 提供引导、`#table` 组织诊断表等即可，不需要 `#crisis` / `#discovery` / `#mastery` 三件套。

## 真 crisis vs 假 crisis 判别准则

`#crisis` 是六阶段链的发动机。如果 crisis 是假的，后面的 discovery、tryit、blueprint 再精彩也是无根之木——读者没有真切感受到"旧工具不够用"，自然不会有"原来如此！"的解脱感。

### 真 crisis 的三个条件

一个真 crisis 必须同时满足：

1. **旧工具实际卡住**：读者拿着前面所有章节学过的工具，*真的*无法解决这个问题——不是"理论上能做，只是没有正式命名"。
2. **卡住点可在具体情境里复现**：困境能落在数轴 / 天平 / 账本 / 地图 / 温度等读者能想象的场景里，不是抽象的"这个概念我们还没定义"。
3. **不只是换名字**：crisis 的结尾让读者心里问"那怎么办"，而不是"哦，这就叫这个名字"。

*例子（真 crisis）*：引入负数时，"账本上已经欠了 3 元，再欠 5 元，总共欠多少"——自然数无法表示"欠债"，旧工具真正失效。

### 假 crisis 的四类反模式

| 反模式 | 识别特征 | 典型例句 |
|---|---|---|
| **命名仪式型** | 读者早就会用这个工具，crisis 只是宣布"现在正式给它起名字" | "以前我们用图表示位置，现在把它叫做'坐标'"——这是 vocabulary，不是危机 |
| **倒灌型** | 先写出 blueprint（工具定义），再往前补 crisis；discovery ≡ blueprint，crisis 是装饰性引子 | discovery 直接给出正式定义，crisis 里没有读者真正被卡住的场景 |
| **答案先泄漏型** | crisis 段已经把新工具的核心思路 hint 出来，discovery 没有新信息 | "笛卡尔发现：只要两条垂直数轴就能确定每个点……"——答案已在 crisis 里了 |
| **尺度过家家型** | 所谓"困境"用旧工具早就能解决，本节并未引入真正的新能力 | "5 个苹果分给 10 个人"——分数早就能解决，没有真正的新需求 |

### 给作者的 5 步自检清单

写完 `#crisis` 之后，逐条核对：

① **拿掉 blueprint 这一节，还能写出 discovery 吗？**
  若 discovery 内容 = blueprint 重述，说明 discovery 是假的，crisis 也是装饰。

② **旧工具具体会卡在哪一步？**
  写下来："读者用 \_\_\_\_ 来尝试，会在 \_\_\_\_ 这一步发现不够用，因为 \_\_\_\_。"
  填不出来 → crisis 可能是假的。

③ **困境能否被一道 tryit 题独立呈现？**
  即"读者还没看 discovery，只凭旧工具，能被这道题真正卡住吗？"
  如果 tryit 题很容易用旧工具做出来，危机不够真实。

④ **history-note 里的原创人，也真的被同一个问题卡过吗？**
  历史人物面对的是实质性困难，不只是"还没给它命名"。
  例：花拉子米写《代数学》，是因为当时没有系统方法处理含未知量的等式约束；不是因为"还没叫它'方程'"。

⑤ **读者读完 crisis 后，心里会问"那怎么办"，而不是"哦，这就叫这个"？**
  前者是真 crisis；后者是词汇课，不是发明链。

## 单节概念预算

每个 `##` 知识点节最多 **3 个真正新学的细分概念**（按读者实际新学的细分概念计数，不允许把多个细分概念打包成一个更大总概念来"凑"到 3 个）。超过 3 个，应拆成多节甚至多章。

## 标准化写作规范（W12）

### 参考答案格式

- 计算 / 应用 / 构造类答案优先写成 `答 / 做法 / 验证` 三段：`答` 给最终结论，`做法` 展示关键步骤，`验证` 用代回、量纲、估算或条件复核确认。
- 证明 / 解释 / 找 Bug 类答案优先写成 `结论 / 理由 / 自查` 三段：`结论` 明确判断，`理由` 给逻辑链，`自查` 回扣题设或常见误判。
- 简短题可把三段压在同一条 `+` 中，但标签要清楚；章末长答案可分行排布。

### 提示卡规范

- `#tryit` 中可以加入 `*提示*：...` 或同等清晰的提示句。提示只指出方法、观察角度或第一步，不泄露最终答案。
- 提示优先帮助读者重新定位旧工具，例如“先画数轴”“先列样本空间”“先写对应顶点表”。

### 难度星级解释

- 章级难度仍只写 `★☆☆☆☆` 到 `★★★★★`，不写具体年级。
- 星级按*概念新度*、*同时调用的工具数量*、*抽象程度*、*证明 / 建模密度*综合判断。
- 大致含义：`★☆☆☆☆` 以直观和单工具为主；`★★☆☆☆` 开始稳定复用旧工具；`★★★☆☆` 有明确新概念并需跨步计算；`★★★★☆` 需要多工具组合、证明或建模；`★★★★★` 需要高抽象、长链推理或跨分支综合。

### 高中桥隔离

- `☞ 高中桥` / `☞ 继续往前` 只作为章末或总结里的可选预告。
- 高中桥内容不进入主线 `#mastery`，不写入 readiness contract，不作为 `book-graph.yaml` 的 prerequisites，也不得成为后续正文推理的必备前置。
- 后续正文若提到高中桥，必须说明它是“可选延伸，不影响主线”。

### `#secref` 粒度规范

- 回查具体概念、工具、公式、判定法时，优先引用 `tool:` 锚点。
- 只有在谈整章背景、整章路线或读者可回到整章系统复习时，才用 `ch:` 锚点。
- 元章节、readiness、总结、自测答案等使用 `meta:` 锚点。

## 章末回顾文件（`99-review.typ`）

每个知识点章节 / 桥梁章节的 `99-review.typ` 统一四段结构：

1. **知识速查卡**（`#blueprint`）：两栏极简总结
2. **混合自测题**（`#mastery`）：跨节综合，不标知识点出处
3. **参考答案**：与混合自测题分开排布
4. **螺旋复习**：2-3 道题显式连接跨章节知识（必须使用 `#secref("tool:...")` / `#secref("ch:...")` 标注），不可混入参考答案段落
5. **可选**："高中桥 / 继续往前" 小节，用 `#side-hack` 给愿意延伸的读者指引；**不计入本章主线 mastery / readiness**，也不写入后续章节或 `book-graph.yaml` 的必备先修。

导读 / 总检 / 终章不套用这套四段，按自身结构写。

## 桥梁章节的额外义务

- 章首页额外说明：这座桥会通向哪些后续分支。
- 每个小节必须显式给出 `☞ 这会在后面哪里再出现`。
- 章末不只是总结，还要给出 `Ready for 代数/几何/函数/数据` 的 readiness contract，固定四段：
  - 已掌握旧工具
  - 最常见误判
  - 回补路径
  - 可以继续前进的信号

## `#tryit` 与 `#mastery` 的自测独立性

`#tryit` 是阅读过程中的主动尝试，`#mastery` 是学完后的自检。两者必须保持结构相似但**数字 / 数据集 / 情境彼此不同**，避免 mastery 直接复述已做过的 tryit 题目。

---

## 核心 / 选做标记

- **Part 知识地图**（`00-part-map.typ`）：表格含 `性质` 列，值为 `核心`（主线必修章节）、`总结`（分支回顾、终章综述）或 `总检`（readiness check）。
- **练习层级**：每个 `#mastery` 块内分三级——`*基础*`、`*应用*`、`*挑战 ☞ 选做*`。基础 + 应用 = 主线掌握标准；挑战 = 可选延伸，后续章节不以它为先修。
- **规则**：后续章节的"需要先会"或正文推理不得依赖任何标记为"选做"的练习所训练的技巧。

### mastery 结构变形（Wave 7 新增）

在不破坏基础 / 应用 / 挑战三层标签的前提下，允许以下三种变形，以缓解长读后的模板疲劳：

| 变形类型 | 说明 | 示例 |
|---|---|---|
| **3 道深题** | 把 5+2+2 改为 3 道多步综合题（每道关联前一道），挑战层保留 1 道高阶题 | 基础：$3 times 17 + 3 times 3$ 用分配律化简，写出计算依据；应用：已知矩形周长 $P$，若长增加 $a$，新面积表达式是什么（展开并整理）；挑战：若长和宽同时增加 $a$，证明新周长一定比原周长大 $4a$ |
| **双解法对比** | 在应用层给 1 道题，要求用方法 A 和方法 B 各做一遍，再写出"哪种更快、为什么" | 题：计算 $999 times 17$。方法 A：直接竖式；方法 B：$(1000 - 1) times 17$。要求：两种方法均写完整步骤，再比较运算量 |
| **推广反推** | 挑战层先给特例，再追问"如果参数变为 $X$ 怎么办" | 特例：已知等腰三角形腰长 $5$、底 $6$，求高；推广：腰长变为 $a$、底变为 $b$，高的表达式是什么？再反问：若高已知为 $h$，能反求腰长吗？ |

**强制保留**：三层标签（`*基础*` / `*应用*` / `*挑战 ☞ 选做*`）必须出现；各层题数不限（基础 ≥ 2，应用 ≥ 1，挑战 ≥ 1）；挑战层依然满足"高阶思维硬规"。

### 挑战层"高阶思维"硬规（Wave 3 新增）

每个 `#mastery` 块的 `*挑战 ☞ 选做*` 层，**至少包含 1 道下列四类之一的高阶思维题**：

| 类型 | 要求示例 |
|---|---|
| **解释 / 论证** | "用自己的话解释为什么……；不得只描述步骤" |
| **证明** | "证明以下命题成立 / 不成立" |
| **反驳 / 找反例** | "举一个具体例子说明……不成立；解释为何它是反例" |
| **找 Bug** | "以下解法有错误，找出错误并说明错在哪里" |

**禁止**：挑战层只增加计算量而不提高思维层次（例如把题目中的数字换大一点，或多加几步同类型计算）。

**合规检查**：每次修改或新建 `#mastery` 块时，先检查是否满足此硬规，不满足则改写或添加。

## 章首入章自检

每个知识点 / 桥梁章节的 `00-overview.typ`，在 `#blueprint` 和 `#side-hack` 之后，用 `#side-hack` 给出 2–3 道入章自检题：

```typst
#side-hack[
  *入章自检*——以下几道小题只用前面章节学过的知识。如果卡住，请先按上面"忘了可以回到"的指引补一补。

  + [只用 prerequisites 的诊断题 1]
  + [只用 prerequisites 的诊断题 2]
  + [只用 prerequisites 的诊断题 3]
]
```

- 题目来源：章首 `*需要先会*` 或 `*前置直觉*` 中列出的知识点。
- 不适用六阶段链的章节（导读 / 总检 / 总结 / 终章）不加入章自检。

## 视觉符号合规

- 禁止 emoji，也不要引入新字体。
- 允许示例符号：`☞`（提示指向）/ `✓`（对）/ `✗`（错）；这是示例而非封闭白名单，低噪点的类似符号也可以少量使用。
- 特别注意：`❌` (U+274C) 是 emoji，不得使用，应写作 `✗` (U+2717)。

## 难度与年级

- **正文与章标题中禁止出现具体年级标签**。章级难度只用 `★☆☆☆☆` … `★★★★★` 星级表达。
- 星级按概念新度、工具数量、抽象程度、证明 / 建模密度综合判断，不按年级贴标签。
- 只允许在少量桥接 / 挑战位置用自然措辞做提示，但不得带"适合 X 年级"的结论性标签。

## 统计范围限制（按中国课标）

**包含**：方差、总体与样本、频率分布直方图、基础概率

**严格排除**：标准差、正态分布、条件概率、独立性检验、排列组合公式、相关系数、回归方程、随机变量、期望值

## `#self-check` 元认知组件

**组件**：`#self-check[问题]`

**视觉**：浅靛色左边框 + `✎ 自检一下` 标签，无底色，紧凑。

**启用规则**：

- `#discovery` 正文字符数超过约 1200 字时，建议在段落中间或末尾插入至少 1 个 `#self-check`。
- `#blueprint` 含 ≥ 3 个新概念（粗体定义词）时，blueprint 之后建议至少插入 1 个 `#self-check`。
- 每个 `#self-check` 内容必须是真实的反思/验证问题，不得套话（如"你明白了吗？"）。

**示范用法**：

```typst
#self-check[
  在继续之前：$3x$ 与 $3x^2$ 是同类项吗？说出你的判断依据。
]
```

```typst
#self-check[
  你刚才看到了"算术平方根"和"平方根"两个说法——它们有什么区别？
  用你自己的话写一句话说清楚。
]
```

**注意**：`#self-check` 必须在文件头部 import 列表中显式引入，例如：
```typst
#import "../../lib/theme-v2.typ": ..., self-check
```

| 类别 | 最低比例 | 说明 |
|---|---|---|
| "小明 / 小红 / 小华 / 小李 / 小王" 等抽象学生角色 | ≤ 30% | 不得超过全书情境人物的三成 |

| 真实工程 / 自然 / 历史场景 | ≥ 30% | 如建筑设计、气象数据、古代历法、地图测量等 |
